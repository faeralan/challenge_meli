# Objetivo

Construir un prototipo de una página de detalle de ítem inspirada en MercadoLibre.  
Además, desarrollar la API backend que la soporte. Este ejercicio evaluará tu capacidad para diseñar e implementar tanto la vista frontend como la funcionalidad backend bajo presión de tiempo.

---

## Requisitos

### 1. Frontend: Página de Ítem

**Diseño y Funcionalidad:**
- Crear una página de detalle de ítem que imite la apariencia de MercadoLibre.
- La página debe mostrar información clave como:
  - Imágenes del producto
  - Título y descripción
  - Precio
  - Métodos de pago
  - Información del vendedor
  - Detalles adicionales (por ejemplo: calificaciones, reseñas o stock disponible)
- No es necesario incluir recomendaciones.
- No se requiere lógica de checkout.

**Experiencia de Usuario:**
- Asegúrate de que la página sea responsiva y fácil de usar.

**Stack:**
- Puedes usar cualquier tecnología o framework frontend de tu elección.

---

### 2. Backend: Desarrollo de API

**Endpoints de la API:**
- Implementar una API RESTful que soporte el frontend proporcionando los detalles necesarios del producto.
- El endpoint principal debe obtener los detalles del producto.

**Stack:**
- Puedes usar cualquier tecnología o framework backend de tu elección.
- No uses bases de datos reales; persiste todo en archivos locales JSON o CSV.

**Requisitos no funcionales:**
- Manejo adecuado de errores.
- Documentación.
- Al menos 80% de cobertura de código.

---

### 3. Documentación

Junto con tu código, incluye un documento breve (1-2 páginas) que:
- Explique tus decisiones de diseño.
- Describa los desafíos enfrentados y cómo los resolviste.

---

### 4. Entrega

- Proporciona un enlace a tu repositorio o un archivo comprimido del proyecto.
- Debe contener un archivo `run.md` explicando cómo ejecutar el proyecto.
